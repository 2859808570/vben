# 系统整体框架

## 一、项目目标

### 概述

- 为需求部门提供全面可追踪的需求池，实现需求的 "提出 - 评审 - 跟踪 – 验收 - 关闭" 的全生命周期管理工具。
- 为开发部门提供精准可执行的需求池，实现需求的 "评审 - 排期 – 开发 – 测试 – 上线" 的全生命周期管理工具。

### 核心目标

- 实现需求规范化、可追溯
- 支持需求优先级排序
- 支持需求变更追踪
- 便于合理分配开发资源、明确开发目标
- 降低因需求变更带来的返工风险
- 提升开发效率和交付质量

## 二、需求流程

### 1、需求正向流程

提交需求 → 需求评审 → 技术评审 → 需求排期 → 功能开发 → 功能测试 → 功能上线 → 功能验收 → 需求关闭

### 2、需求变更流程

**未开发的需求：**

撤回需求 → 变更需求 → 重新提交

**开发中的需求：**

申请撤回 → 撤回审批 → 变更需求 → 重新提交

### 3、需求作废流程

**未开发的需求：**

需求作废 → 需求关闭

**开发中/已开发的需求：**

申请作废 → 作废审批 → 需求作废 → 功能下线（如已上线）→ 需求关闭

## 三、适用角色

### 需求提交人

**使用场景：**

- 创建新需求，填写需求基本信息、业务背景、优先级、预期完成时间等
- 跟踪需求状态，了解需求在流程中的进展情况
- 对已上线的功能进行验收，确认是否满足原始需求
- 当需求需要调整时，申请需求变更或撤回
- 对不再需要的需求申请作废

### 需求评审人

**使用场景：**

- 对提交的需求进行评审，评估需求的合理性、可行性和优先级
- 审核需求文档的完整性和准确性
- 参与需求变更的评审，评估变更的影响和必要性
- 参与需求作废的审批，确认作废的合理性

### 项目负责人

**使用场景：**

- 根据资源情况和优先级，对需求进行排期和分配
- 协调开发资源，确保需求按时交付
- 跟踪项目进度，管理需求开发状态
- 审批需求变更和作废申请
- 监控需求完成情况，进行项目总结和复盘

### 软件工程师

**使用场景：**

- 接收分配的需求，查看需求详细信息和开发要求
- 进行功能开发和代码实现
- 更新需求开发进度和状态
- 参与技术评审，评估技术可行性
- 处理需求变更，调整开发方案

### 测试人员

**使用场景：**

- 查看需求文档，编写测试用例
- 对开发完成的功能进行测试，确保功能符合需求规格
- 记录测试结果和缺陷，反馈给开发人员
- 参与需求验收，确认功能质量
- 跟踪缺陷修复情况，进行回归测试

### 软件团队管理人员

**使用场景：**

- 查看需求统计报表，了解团队工作量分布
- 分析需求完成情况，评估团队效率
- 管理需求优先级，优化资源配置
- 审批重要需求的变更和作废
- 进行需求数据分析，为决策提供支持

## 四、项目模块分布

### 需求管理模块

负责需求的全生命周期管理，包括需求的创建、评审、排期、开发、测试、验收、关闭等流程。

**主要功能：**

- 需求创建与编辑
- 需求评审管理
- 需求状态跟踪
- 需求变更管理
- 需求作废管理
- 需求查询与统计

### 项目管理模块

负责项目的创建、规划、执行和监控，支持多项目并行管理。

**主要功能：**

- 项目创建与配置
- 项目计划制定
- 项目进度跟踪
- 项目资源分配
- 项目风险管控
- 项目报表分析

**项目阶段：**

- **项目立项**：项目启动阶段，包括项目可行性分析、项目目标确定、项目章程制定、项目团队组建等。在此阶段明确项目范围、时间节点和关键里程碑。

- **项目开发**：项目执行阶段，包括需求分析、系统设计、编码实现、单元测试等。在此阶段按照项目计划进行功能开发，跟踪开发进度，管理开发任务和代码版本。

- **项目测试**：项目质量保证阶段，包括测试计划制定、测试用例编写、功能测试、集成测试、性能测试等。在此阶段发现和跟踪缺陷，确保功能质量和系统稳定性。

- **项目上线**：项目部署阶段，包括上线准备、环境部署、数据迁移、功能验证、项目Review等。在此阶段完成系统部署，确保生产环境正常运行，并进行上线后的监控和问题处理。

- **项目验收**：项目收尾阶段，包括功能验收、文档交付、培训支持、项目总结等。在此阶段确认项目成果符合预期，完成项目交付，并进行项目复盘和经验总结。

### *项目升级模块

负责项目小版本的全生命周期管理，包括版本规划、需求归集、版本发布等，支持项目的快速迭代和持续优化。

**主要功能：**

- 版本创建与规划
- 版本需求归集与管理
- 版本计划制定（小版本快速迭代）
- 版本进度跟踪
- 版本发布管理
- 版本历史追溯

**版本管理流程：**

- **版本规划**：项目上线后，收集用户反馈和新需求，进行小版本规划，确定版本号（如 v1.1、v1.2等）、版本目标和计划发布时间。小版本通常周期较短，聚焦于增量功能更新。

- **需求归集**：将用户提出的新需求归集到对应小版本中，根据优先级和版本容量进行需求筛选和排序。小版本通常包含少量核心需求，确保快速交付。

- **版本开发**：按照小版本计划进行需求开发，跟踪版本开发进度。小版本开发周期短，注重快速实现和交付。

- **版本测试**：对小版本功能进行测试，重点验证新功能与现有功能的兼容性，确保版本质量。

- **版本发布**：完成版本测试后，进行小版本发布，包括版本部署、功能验证、发布通知等。小版本发布频率较高，注重快速响应和持续优化。

- **版本维护**：版本发布后，持续收集用户反馈，处理版本问题，为下一小版本规划提供参考。

**与项目管理模块的关系：**

版本管理模块与项目管理模块相互关联，小版本可以作为项目的迭代进行管理。小版本遵循简化的项目流程，但版本管理更侧重于多小版本快速迭代的规划和协调，而项目管理更侧重于单个项目/版本的执行和监控。小版本迭代通常不需要完整的项目立项流程，而是基于现有项目进行快速迭代。

### 功能开发模块

负责功能开发过程的管理，包括开发任务分配、进度跟踪、代码管理等。

**主要功能：**

- 开发任务分配与跟踪
- 开发进度更新
- 代码提交与版本管理
- 技术文档管理
- 开发问题记录与处理
- 开发工作量统计

### 功能测试模块

负责功能测试过程的管理，包括测试计划制定、测试用例管理、缺陷跟踪等。

**主要功能：**

- 测试计划制定
- 测试用例编写与管理
- 测试执行与结果记录
- 缺陷提交与跟踪
- 测试报告生成
- 测试覆盖率统计

### 用户权限管理模块

负责系统用户的管理和权限控制，确保不同角色用户拥有相应的操作权限。

**主要功能：**

- 用户账号管理
- 角色定义与分配
- 权限配置管理
- 组织架构管理
- 登录认证与安全控制

### 系统配置模块

负责系统基础配置和参数设置。

**主要功能：**

- 系统参数配置
- 工作流配置
- 通知规则配置
- 数据字典管理
- 系统日志管理
