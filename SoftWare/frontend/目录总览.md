# Frontend 目录总览

目录使用 Monorepo（将多个项目放在一个仓库中统一管理）管理，项目结构如下：

```bash
frontend/
├── .browserslistrc # 浏览器兼容性配置，指定项目需要支持的浏览器版本
├── .changeset # Changeset 版本管理，用于管理版本号和更新日志的工具
│   ├── config.json # Changeset 配置文件
│   └── README.md # Changeset 使用说明
├── .commitlintrc.js # Commitlint 配置文件，检查提交信息格式是否符合规范
├── .dockerignore # Docker 忽略文件，指定构建镜像时忽略的文件
├── .editorconfig # 编辑器配置，统一不同编辑器的代码格式
├── .gitattributes # Git 属性配置，控制 Git 如何处理文件
├── .gitconfig # Git 配置文件
├── .github # GitHub 配置目录，存放 GitHub 相关的配置和工作流
│   ├── actions # GitHub Actions，自动化工作流脚本
│   │   └── setup-node # Node.js 设置 Action，用于设置 Node.js 环境
│   ├── CODEOWNERS # 代码所有者，指定代码审查负责人
│   ├── commit-convention.md # 提交规范文档，说明如何编写提交信息
│   ├── config.yml # GitHub 配置文件
│   ├── contributing.md # 贡献指南，说明如何参与项目开发
│   ├── dependabot.yml # Dependabot 配置，自动检查依赖更新
│   ├── ISSUE_TEMPLATE # Issue 模板，创建 Issue 时的表单模板
│   │   ├── bug-report.yml # Bug 报告模板
│   │   ├── docs.yml # 文档模板
│   │   └── feature-request.yml # 功能请求模板
│   ├── pull_request_template.md # PR 模板，创建 Pull Request 时的模板

│   └── workflows # GitHub Actions 工作流，自动化 CI/CD 流程
│       ├── build.yml # 构建工作流，自动构建项目

│       ├── deploy.yml # 部署工作流，自动部署到服务器
│       ├── draft.yml # 草稿工作流
│       ├── issue-close-require.yml # Issue 关闭要求，关闭 Issue 时的检查

│       ├── lock.yml # 锁定工作流，锁定依赖版本
│       ├── release-tag.yml # 发布标签工作流，自动创建发布标签
│       ├── rerun.yml # 重新运行工作流
│       ├── semantic-pull-request.yml # 语义化 PR 检查，检查 PR 标题格式
│       └── stale.yml # 过期 Issue 处理，自动标记长期未处理的 Issue
├── .gitignore # Git 忽略文件，指定 Git 不跟踪的文件
├── .gitpod.yml # Gitpod 配置，云端开发环境配置
├── .node-version # Node.js 版本，指定项目使用的 Node.js 版本
├── .npmrc # npm 配置，npm 包管理器的配置
├── .prettierignore # Prettier 忽略文件，指定不格式化的文件
├── .prettierrc.mjs # Prettier 配置文件，代码格式化规则
├── .stylelintignore # Stylelint 忽略文件，指定不检查样式的文件
├── .vscode # VS Code 配置，编辑器工作区配置

│   ├── global.code-snippets # 代码片段，自定义代码快捷输入
│   ├── launch.json # 调试配置，调试运行配置
│   └── settings.json # 工作区设置，编辑器设置
├── apps # 项目应用目录，存放实际运行的应用

│   │   ├── api # API 路由定义，定义模拟的 API 接口

│   │   │   └── user # 用户 API，用户相关接口

│   └── web-antd # 基于 Ant Design Vue 的前端应用，项目的主要前端应用（详细目录结构请查看 web-antd目录说明.md）

├── docs # 项目文档目录，存放项目文档（使用 VitePress 构建）

├── internal # 内部工具目录，项目内部使用的工具和配置
│   ├── lint-configs # 代码检查配置，代码质量检查工具的配置
│   │   ├── commitlint-config # Commitlint 配置，提交信息格式检查配置

│   │   ├── src # 源代码

│   │   │   ├── prettier.ts # Prettier 工具，代码格式化工具
│   │   │   └── spinner.ts # 加载动画，命令行加载提示
│   │   └── tsconfig.json # TypeScript 配置
│   ├── tailwind-config # Tailwind 配置，Tailwind CSS 通用配置
```
